## Architecture

**App**: Next.js 14 App Router + TypeScript. Route groups:

| Group | Path | Auth |
|-------|------|------|
| `(auth)` | Login, register, magic-link landing | public |
| `(player)` | Dashboard, round scoring, leaderboard, scorecard | player session |
| `(admin)` | Tournament, players, teams, holes, clubs, sponsors, scores | admin role |
| `live/[slug]` | Public real-time leaderboard | none |
| `api/` | `POST /api/shots`, `POST /api/magic-link` | varies |

**Data flow**:

```
Player phone
  └─ Next.js browser
       ├─ SyncEngine (localStorage queue) ──► Supabase REST  ──► DB
       ├─ GPS (navigator.geolocation + Haversine)
       └─ useRealtimeScores (5s debounce) ◄── Supabase Realtime
                                                  └─ calculate-best-ball (Deno Edge Function)
```

**Key lib modules** (`src/lib/`):

- `types.ts` — all 9 entity interfaces (`Tournament`, `Team`, `Player`, `Shot`, `Score`, etc.)
- `scoring.ts` — `buildTeamCard()`, `totalVsPar()`, `formatVsPar()` (Best Ball logic, server-side safe)
- `sync-engine.ts` — offline write queue singleton; max 5 retries per item; persists to `localStorage`
- `gps.ts` — `GpsPosition` (uses `lat`/`lng` not `latitude`/`longitude`), `getCurrentPosition()`, `distanceMeters()` (Haversine)
- `supabase/client.ts` / `server.ts` — browser and SSR Supabase clients via `@supabase/ssr`

**Supabase** (`supabase/`):

- `migrations/001_initial_schema.sql` — 9 tables, RLS, realtime; deferred captain FK (circular reference)
- `migrations/002_leaderboard_rpc.sql` — `get_leaderboard()` RPC function
- `migrations/003_tournament_pause_state.sql` — adds `'paused'` to tournament status enum
- `functions/calculate-best-ball/` — Deno Edge Function; uses service role key to bypass RLS
- `seed.sql` — CIBC tournament, 21 clubs, 18 Granite Ridge holes

## Non-Obvious Technical Decisions

| Decision | Reason |
|----------|--------|
| `supabase/functions` excluded from `tsconfig.json` | Deno CDN imports (`https://esm.sh/...`) break the Next.js TS compiler |
| 5s debounce on `useRealtimeScores` | Prevents 125-client Supabase channel storm on a single leaderboard update |
| `SyncEngine` write-queue (max 5 retries) | Offline-first: survives page refresh; drops items after 5 failures to prevent infinite retry |
| Deferred `fk_teams_captain` FK via `ALTER TABLE` | `teams` and `players` have a circular FK; FK added after both tables exist |
| Mapbox via `react-map-gl` + `mapbox-gl` | `NEXT_PUBLIC_MAPBOX_TOKEN` — satellite view for shot tracking and pin editor |
| `docs/dashboard.html` in `.gitignore` | Auto-generated by `tools/watch-dashboard.js`; causes constant merge conflicts |

## Testing

Tests live in `src/__tests__/`. Coverage is collected only from `src/lib/**/*.ts` and `src/app/api/**/*.ts` — Supabase SSR wrappers and `utils.ts` are excluded (no logic).

Thresholds (enforced in CI): **≥80%** statements/functions/lines, **≥70%** branches.

API route tests must declare `@jest-environment node` at the top of the file — Next.js route handlers use the Web `Request` API, which is unavailable in jsdom.

## Git Workflow

```
feature/US-XXXX-short-name   → squash-merge into develop via PR
bugfix/BUG-XXXX-short-name   → squash-merge into develop via PR
release/X.Y.Z                → merge into main via PR
hotfix/BUG-XXXX-short-name   → branch from main, merge into main + develop
```

Both `main` and `develop` are **protected** — never push directly to either branch.

**Commit format:**
```
[TYPE] SHORT-ID: Short imperative description (max 72 chars)
```
Types: `feat`, `fix`, `test`, `docs`, `refactor`, `chore`, `style`, `perf`

Pre-commit hook (Husky + lint-staged) runs Prettier + ESLint on `src/**/*.{ts,tsx}`.
