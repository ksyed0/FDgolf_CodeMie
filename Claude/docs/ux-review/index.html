UX Walkthrough
Tournament golf score tracking · Web · Mobile-first
Prepared for external Product Owner review · 2026-06-08
First production tournament
CIBC ARC Golf — 22 June 2026, Granite Ridge GC

Format: Best Ball, 18 holes, shotgun start

Field: ~125 players in 32 teams

Run mode: Browser-based on mobile (iOS & Android); admin tools also work on tablet and desktop

Strategy
FDgolf is a multi-tournament platform. The CIBC event proves the model end-to-end so subsequent tournaments scale without rebuilds.
Frontend
Next.js 14 App Router
SSR for public & admin views, Client Components for active round tracking, PWA-shell behaviour for offline tolerance.
Backend
Supabase
PostgreSQL with Row Level Security, Supabase Auth, Realtime channels for live leaderboard, RPC functions for scoring.
Mapping
Mapbox GL JS
Custom golf style + 3-layer aggressive caching (course tiles, hole snapshots, client-side haversine distance).
Offline
IndexedDB queue
Graceful degradation — shots write locally and sync on reconnect with newer-wins conflict resolution.

Journey 3 · Round Tracking
Outcome Picker
  • Four large colour-coded buttons — green Sunk (success), red OOB (danger), amber Mulligan (caution), neutral-green In-Play (continue).
  • Help legend in the grey card explains each — first-time users don't have to guess.
  • OOB outcome triggers a follow-up prompt: rehit from the OOB spot or from the prior shot's origin? The data model links the rehit shot back via rehit_from_shot_id.

Journey 5 · Admin
Score Editor + Audit
  • Per-shot editor — club, outcome, GPS, distance all editable. Hole and team scores recompute via DB trigger.
