**Epic:** EPIC-0001 Foundation & Infrastructure
**Status:** Design (in progress)
**Author:** Compass
**Depends on:** None

## 1. Goal

This story creates the shared foundation every other epic builds on. A working Next.js 14 App Router project with TypeScript strict mode, Tailwind CSS, shadcn/ui, and Supabase client factories must exist before any feature work can start. Without this scaffold, no agent can begin implementation — it is the first story and must be completed before any parallel work is attempted.

The scaffold should boot cleanly, prove each dependency is wired, and establish the folder conventions all subsequent stories will follow (`app/`, `components/ui/`, `lib/supabase/`).

## 2. Acceptance Criteria

- AC-0001: Next.js 14 App Router project boots locally with `npm run dev` and renders a placeholder page
- AC-0002: TypeScript strict mode enabled; `tsconfig.json` extends Next.js defaults
- AC-0003: Tailwind CSS configured with PostCSS pipeline; sample utility class renders correctly
- AC-0004: shadcn/ui CLI initialized; `components/ui/` directory exists with at least `button.tsx` installed
- AC-0005: `@supabase/ssr` and `@supabase/supabase-js` installed; browser and server client factories exist in `lib/supabase/client.ts` and `lib/supabase/server.ts`
- AC-0006: `.env.local.example` documents `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `NEXT_PUBLIC_MAPBOX_TOKEN`

## 3. Out of Scope

- Supabase project provisioning or cloud configuration (US-0002)
- App chrome / header component or branding (US-0003)
- Auth middleware or login flows (US-0004)
- Database schema or migrations (US-0005)
- RLS policies (US-0006)
- Mapbox map rendering (US-0007)
- Club seed data (US-0008)
- Any feature-level pages or components beyond the placeholder

## 4. Technical Notes

- The Next.js app should live in a subdirectory named `app-src/` (or similar) within the `Claude/` monorepo folder — do not place it at the root to avoid conflicting with the existing `package.json` (PlanVisualizer tooling). Keystone (Architect) will confirm the exact nesting in the Technical Design section.
- `.env.local` must NOT be committed; only `.env.local.example`.
- The Supabase server client must use `@supabase/ssr` cookies helper for SSR compatibility.

## Technical Design

**Author:** Keystone

### Repository placement

The Next.js application lives at `Claude/fdgolf-app/` inside the `Claude/` subfolder. This keeps the FDgolf app co-located with its project management files while keeping the monorepo root clean for independent apps (e.g. `CodeMie/`).

```
FDgolf/
  .git/
  .gitignore
  Claude/              ← FDgolf app + PlanVisualizer project management
    fdgolf-app/        ← Next.js 14 App Router application  ← THIS STORY
      app/
        layout.tsx
        page.tsx
        globals.css
      components/
        ui/            ← shadcn/ui primitives (button.tsx etc.)
      lib/
        supabase/
          client.ts    ← browser Supabase client
          server.ts    ← server-side Supabase client (SSR cookies)
      public/
      .env.local.example
      next.config.ts
      tailwind.config.ts
      tsconfig.json
      package.json     ← Next.js deps (separate from Claude/package.json)
      postcss.config.mjs
  CodeMie/             ← independent app (separate project)
```

### Key file contents

**`lib/supabase/client.ts`** — browser client (Client Components, event handlers):
```typescript
import { createBrowserClient } from '@supabase/ssr'

export const createClient = () =>
  createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )
```

**`lib/supabase/server.ts`** — server client (Server Components, Route Handlers, middleware).
Uses Next.js 14 synchronous `cookies()` API with `get`/`set`/`remove` (NOT the Next.js 15 async `getAll`/`setAll` pattern):
```typescript
import { createServerClient, type CookieOptions } from '@supabase/ssr'
import { cookies } from 'next/headers'

export function createClient() {
  const cookieStore = cookies()
  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        get(name: string) {
          return cookieStore.get(name)?.value
        },
        set(name: string, value: string, options: CookieOptions) {
          cookieStore.set({ name, value, ...options })
        },
        remove(name: string, options: CookieOptions) {
          cookieStore.set({ name, value: '', ...options })
        },
      },
    }
  )
}
```
