- [ ] **Step 2: Create `map-view.tsx`**

  Create the file at `/Users/Kamal_Syed/Projects/FDgolf/Claude/fdgolf-app/components/map-view.tsx` with the following content:

  ```tsx
  'use client'

  import Map from 'react-map-gl'
  import 'mapbox-gl/dist/mapbox-gl.css'

  const DEFAULT_STYLE_URL =
    process.env.NEXT_PUBLIC_MAPBOX_STYLE_URL ??
    'mapbox://styles/mapbox/outdoors-v12'

  export interface MapViewProps {
    lat: number
    lng: number
    zoom?: number
    styleUrl?: string
  }

  export default function MapView({
    lat,
    lng,
    zoom = 15,
    styleUrl,
  }: MapViewProps) {
    const token = process.env.NEXT_PUBLIC_MAPBOX_TOKEN

    if (!token) {
      return (
        <div
          className="flex items-center justify-center rounded-lg border border-dashed border-gray-300 bg-gray-50 p-8 text-center text-sm text-gray-500"
          role="alert"
          aria-label="Map unavailable"
        >
          <p>Map unavailable — set NEXT_PUBLIC_MAPBOX_TOKEN to enable the map.</p>
        </div>
      )
    }

    return (
      <Map
        mapboxAccessToken={token}
        initialViewState={{
          longitude: lng,
          latitude: lat,
          zoom,
        }}
        style={{ width: '100%', height: '100%' }}
        mapStyle={styleUrl ?? DEFAULT_STYLE_URL}
      />
    )
  }
  ```

  Key design points:
  - `'use client'` directive at line 1 — required because `mapbox-gl` uses browser APIs (WebGL, DOM).
  - The CSS import (`mapbox-gl/dist/mapbox-gl.css`) must live inside the Client Component so Next.js bundles it for the browser only.
  - `DEFAULT_STYLE_URL` reads `NEXT_PUBLIC_MAPBOX_STYLE_URL` env var first, falling back to the `outdoors-v12` style (appropriate for golf courses). Callers can override per-instance via the `styleUrl` prop.
  - Token guard returns a styled fallback `<div>` — never throws, never renders a broken `<Map>`.
  - `initialViewState` (not `viewState`) is used so the map remains uncontrolled by default; callers can upgrade to controlled mode in later stories.

      // Reset env before each test

        render(<MapView lat={37.5} lng={-122.4} />)
        expect(screen.queryByTestId('mapbox-map')).not.toBeInTheDocument()
      })
    })

    describe('when NEXT_PUBLIC_MAPBOX_TOKEN is present', () => {
      beforeEach(() => {
        process.env.NEXT_PUBLIC_MAPBOX_TOKEN = 'pk.test-token-abc'
      })

      it('renders the mocked map element', () => {
        render(<MapView lat={37.5} lng={-122.4} />)

      it('forwards a custom zoom prop', () => {
        render(<MapView lat={37.5} lng={-122.4} zoom={12} />)
        expect(screen.getByTestId('mapbox-map')).toHaveAttribute('data-zoom', '12')

      it('uses the outdoors-v12 default style when styleUrl is omitted', () => {
        delete process.env.NEXT_PUBLIC_MAPBOX_STYLE_URL
        render(<MapView lat={37.5} lng={-122.4} />)

          'data-style',
          'mapbox://styles/custom/golf'

## Self-Review Notes (Keystone)

1. **Spec coverage:** All 5 ACs (AC-0035 through AC-0039) have an explicit verification step. Each is traceable to a specific task.

2. **Client Component constraint:** `'use client'` is the first line of `map-view.tsx`. The CSS import lives inside the Client Component module so it is never evaluated server-side. No Server Component ever directly imports `MapView` — callers that are Server Components must wrap it in a Client Component boundary or import it into a file already marked `'use client'`.

3. **WebGL / jsdom incompatibility:** Addressed in TASK-0284 by mocking `react-map-gl` entirely. The mock renders a `data-testid="mapbox-map"` `<div>` with all relevant props as `data-*` attributes, enabling full prop-forwarding assertions without any WebGL calls.

4. **Token env var strategy:** `process.env.NEXT_PUBLIC_MAPBOX_TOKEN` is read at render time inside the component body (not at module level), so the Vitest `vi.stubEnv` / `process.env` mutation approach works reliably. If the build inlines the value (which Next.js does for `NEXT_PUBLIC_*` in production), the fallback is still safe — an absent token in production becomes an empty string, which is falsy.

5. **Style URL configurability (AC-0037):** Two layers of configurability are provided — the `NEXT_PUBLIC_MAPBOX_STYLE_URL` env var sets a project-wide default; the `styleUrl` prop overrides it per-instance. The sensible default (`outdoors-v12`) is appropriate for golf course terrain.

6. **CI workflow:** The `ci.yml` file uses `working-directory: CodeMie` as a default. The Build step's `env` block is the correct place for `NEXT_PUBLIC_MAPBOX_TOKEN` — it does not affect the working directory default.

7. **TDD ordering:** TASK-0284 tests are written against the component created in TASK-0283. Steps explicitly say to run tests before checking coverage, ensuring tests are not silently skipped.

8. **Commit cadence:** Four commits (one per TASK-0282 through TASK-0285) + one push in TASK-0286. TASK-0286 is verification-only.
